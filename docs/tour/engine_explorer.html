The trusted engine —
and why we test it.

Every result in RoughVolLab is built on one path generator: roughvol_core.py, the κ=0 Volterra engine. It exists because an earlier engine had a silent bug — its variance was normalised wrong, so any analysis built on it would measure the bug, not the market. This tool lets you drive the real engine, and see the bug it was built to prevent.

Advanced — the bug, and the test that catches it

WHY IT'S TESTED show ▼

The Volterra process has a variance that grows over time. The correct way to normalise the model uses the discrete variance the scheme actually produces — call it v_i. An earlier engine used the continuum formula t^(2H) instead, which is subtly wrong for a discretised path: it assumes the process has unit variance, when the discretised path’s true variance is smaller — around 15% smaller at H=0.10 — a gap that quietly biases everything built on it. The paths still looked fine. Flip the bug on below, and run the test.

This is the actual test in test_roughvol_core.py — test_volterra_variance_matches_discrete_formula. It draws thousands of paths and checks the empirical variance against the discrete formula, failing if they diverge. With the bug on, the paths look normal but the test fails — which is exactly the point: the error is invisible to the eye, and only checking against the known answer catches it. That is why the whole library imports from this one tested module, and never from the engine with the bug.
